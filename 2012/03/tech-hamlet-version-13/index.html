Say Hello to Our Brand New Tech Hamlet ( Version 13.0 )

Mar 4th, 2012 | Comments

Our previous version Tech Hamlet 12.0 is a completely unique design. But, we made it in a big hurry! We wanted to switch to a new host before the previous one expired. So, it’s a big hurry and we had to waste a lot of money to hire people and configure the server. We went into a lot of trouble making that change. Just like that, it also gave us a lot of troubles in the past few months. There were some design flaws. We thought of fixing them later but it never happened. The theme framework we used (Thesis) didn’t got enough updates and it started to conflict with the plugins. The software running in the server also got very old and some conflicts start to rise in it. So, we decided to go for the biggest update we made in our history!

This time, we took as much time as we can and tried to make things perfect as possible. We tested things in our local machines and then installed on the new server and again tested them. Fixed bugs, tuned the server, made minor adjustments and now we are confident about our new site! Lets see what we did one by one…

A new theme framework and a new design :

Earlier, we used Thesis. A popular theme framework with a lot of SEO support. But, it had several problems. The biggest issue is that it refused to folow the Wordpress standards. Because of this, some plugins started to give errors. Some never even worked. It also had conflicts with my caching plugins and tried to mess up the databases several times. The other issue is that Thesis is not updated frequently. Its getting old. They say that they are going to release Thesis 2.0. But where is it? It’s too late…

We switched to Genesis. A well updated theme framework developed by StudioPress. It stays with Wordpress standards and its even easier to customize and update the theme. We won’t lose the design if we update the core framework! That’s really cool!

We changed our framework and then started to build our own theme. On January, we introduced our second site Cyber Gnome. Many people liked its design because it was very simple. So, I thought of putting some of those features into TH. So, you can see that TH and CG have slight similarities. But TH is different. Cyber Gnome have several unique features tailored to its theme. For example, it’s a news site… We need to add a lot of new topics to it. So, the reader must see how fresh the article is. That’s why we have made the published time more visible. It will tell you how many minutes have passed since we published the post. But TH doesn’t need such things.

Earlier when I made the designs, I didn’t think much about development rules and didn’t even write clean code. But its different in TH 13. The code is clean and well commented.

Since we do a lot of giveaways, we have added a giveaway box to the top of the sidebar. In TH 12 also we did this. But we had to put it manually. In TH 13, it will be added automatically and it will also get removed when we end the giveaway.

We also have widgets to show latest news from Cyber Gnome, to show the top stories in the home page and to show latest stories in the posts page (Those two stories widgets were ideas of Shanaka Jayatilake).

This time, we moved the social counts to the very top and did some adjustments to the code so that it will load more faster. Now it doesn’t need to get the RSS Feed count every time a user load the page… We have used the Wordpress cron to get the RSS count and  cache it twice a day.

Now The home page is clean… We changed the format of our posts. Earlier, we had different sizes of images and paragraphs in the home page. Now its well-organized. Earlier we adjusted the home page content with a read more tag. But now its auto generated. All the images are of the same dimension… Its neat! In the post page also you’ll notice that things are neatly organized.

Less ads :

I don’t know about you but I really hate ads. One of the biggest reasons we dropped TH 12 is its full of ads. When you come to a post, there’s an ad at the top, to the left of the post and at the very end of the post. The header ad also didn’t go with the theme. But this time, we have a big ad in the top, top of the sidebar, in the author bio and a small set of text links at the end of the post. No ads will obstruct the post. We are planing to get rid of those 125 banners as soon as we make sure our new advertising plan is working. A new ad plan? Its a big story! I’ll tell you about it tomorrow… (It’s a science fiction story!)

New server software but the same host :

If you stayed with us for the last year, you should already know that we had some difficulties in finding a good VPS host for our site. I think we tried more than 5 hosts. Some we tested for months but they didn’t work for us. We were afraid to go for self managed hosts. But, finally we went for Media Temple and hired some experts to configure it. We did this change while releasing our previous version 12.0. But I think Media Temple is the right host for us. Their servers are really great. A lot of CPU power… But, I can’t say much about their support because I don’t use it much. The server is self managed so I take care of things by myself.

But soon we recognized that the OS and the software we installed on it were really old. It also had several conflicts. So, we decided to re install everything. We took our site to a shared server temporarily and made the changes. I did some of the stuff by myself and hired a freelancer to check things up. I must say the new server is performing really well and now even my cache plugins are working better than before. Earlier the css and js minification never worked. But now, it works perfectly! No idea whether it’s the server or the theme Framework…

Back to the old commenting system Disqus. But we did some cool changes…

Recently, we tested Livefyre on our site. Its a real-time comment system which really worked. That system is really great but it’s still new and had several stability issues. It didn’t write the comment back to my database for several weeks. So, I contacted them and tried to fix things. It looks like things are going to get delayed. But I should say that their support is really great and they’ll try to help you in any way they can. Will consider about it in the future! (Our site Cyber Gnome still use Livefyre).

Finally, we switched back to our old favourite Disqus! Its more stable and easy to moderate. Normally, Disqus allows you to customize things with custom CSS… But I never used it… This time, we changed it to fit the theme of our site. Check it out and let me know what you think!

We cleaned up a lot of junk in our databases and made a lot of core improvements…

With time, your WordPress databases can grow big. Some old and removed plugins might have left some junk behind. When moving to our new server, I scanned and removed a lot of junk like this. Not only the post meta tables, but also the post table got a lot of junk. If you want to prune it, check for posts with post type set to something other than the standard types. I cut down the size of my post database by half. The thing is, I used Intense debate earlier. It have written a lot of entries to my posts tables. A lot! Lucky for me, Livefyre gave me issues. That’s why I took a look at the databases… Now they were removed and the site is still working!

This time we re-installed Wordpress and also added a new set of plugins. Even replaced some of the plugins with our own code snippets.  As I said earlier, we built the system in a local test machine and after making sure its stable enough to work on the live site, we installed it. Then, took more than a week, fixing bugs, adjusting the theme, tuning the server to handle traffic and use the resources more efficiently, and tuning the whole site. We never did this before. After we put up the design, we normally forget about the problems which come next. This time, we went searching for them and took them down.

We also made a lot of core improvements. Some are from the server-side. For example, normally, the Wordpress cron will be executed every time a user visit the site. This will add too much load to the server. It also made our backup plugins go crazy. So, we prevented it from running automatically and set the server’s cron to execute it every 5 minutes. Now things are perfect.

We learnt a lot while putting up this new version. We will surely share them with you in our upcoming posts. So, open your eyes and stay tuned!

So, what do you think? Don’t you think this new version is better than the others? Don’t you think that we can take our site to the next level with this update? Leave your comments and suggestions in the comment section below. Also if you have any critics, post them also! We need to know your true feelings about our new site!
