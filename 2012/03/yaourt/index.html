Arch Linux : Install Any AUR Package Easily With the Pacman Alternative Yaourt

Mar 24th, 2012 | Comments

In Ubuntu, if you want to get a package which is not in the official repository, if you want to get the latest version of a package from another repository, we can use PPAs. A lot of third-party users have created their own repositories in Launchpad and we can add these PPA’s to easily get those packages through our package manager.

In Arch Linux, we don’t want to go for third-party repos to get the latest versions. They always keep their repository updated. But what if the package is not available in this repository? Then we have to check the Arch User Repository (AUR) which contain packages maintained by third-party developers.

If we want to install a package from the AUR, we can’t just add the source and use our default package manager pacman to install them. We have to manually download the package, build it, resolve any dependencies and then install it. If that package have many dependencies that we have to get from the AUR, then you’ll get really frustrated trying to install all of them. But, things will be much easier if we had a package manager for AUR right? That’s why we have Yaourt!

Yaourt is a package manager just like pacman. Actually it adds more functionality to pacman. Now, to install a package from the AUR, you don’t have to download the tarballs and resolve dependencies manually. Yaourt will do it for you. If you want to use this tool, you don’t have to learn any extra syntax. It have the same syntax of pacman. Cool isn’t it?
