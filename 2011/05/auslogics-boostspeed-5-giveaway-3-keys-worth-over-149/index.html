When I first heard it I also thought this might a be cheap trick. Honestly I know nothing of TCP or Networking. So I decided to hook up to the Wikipedia. Then I understood why this actually work and there is such a thing as Internet Optimization.

When a computer connects to the internet, there is a loss of data packets when transferring data between the server and the computer. What BoostSpeed does adjust these options to the optimal level. The main too 3 options that influence the data packet transfer are

  • MTU(Maximum Transmission Unit Size) Greater the MTU value more the data can be transported to the user. This value is properly handled by BoostSpeed.

  • RWIN(TCP Receive Window) The maximum amount of data that can be absorbed without  acknowledging the server. If the server is not acknowledged of the data it will retransmit the data again and this will cause loss of packet data. Optimizing the RWIN value will reduce the loss of service data and packet re-send data.

  • TTL(Time to live for TCP/IP packets) TTL value resembles the time a packet data can experience on the Computer network before it’s discarded. Connection interruptions can be minimized by optimizing this value.
