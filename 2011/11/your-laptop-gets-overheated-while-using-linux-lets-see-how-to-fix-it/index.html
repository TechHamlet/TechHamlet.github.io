Your Laptop Gets Overheated While Using Linux? Lets See How to Fix It!

Nov 18th, 2011 | Comments

If you own a new Laptop and if you tried to install Linux on it, you might face a big problem. You might notice that your machine gets overheated madly. Some of you can even boil some water with your laptop! Why is this happening? Is it a problem with the hardware or is it a bug in Linux? Can we fix it? If we can, how to fix it?

This is a huge problem which affects most of the Linux users who use mobile devices. Some say that the biggest problem is with the Power Management in the Linux kernel. ASPM (Active State Power Management) is used to save power in PCI Express links by setting a lower power state for idle devices. In Linux kernel 2.6.38, ASPM will be switched off if the BIOS reports that it doesn’t support it. But, unfortunately, many BIOSes have misconfigured these settings so, even though the system support ASPM, it will be switched off! So, your machine draws a lot of power and runs at full speed even when it’s not required. This is not good for mobile devices…

Now, lets see how to fix this problem… I won’t tell you how to fix it step by step (Things are not that easy this time). There are a lot of articles around the web which gives you several solutions for this problem. Lets see whether they can help us.

I have a Core i7 HP Pavilion DV6 Laptop. In Windows, the normal CPU temperature stays between 55 and 58. But, when I install Linux (I tried Ubuntu), the temperature stays between 78 to 80! A few more and I can boil some water. Today I tried various things to fix this problem :

1) Recently, Matthew Garrett have created a kernel patch which might fix the ASPM issue. So, our developers at Ubuntu have compiled a special kernel with this patch. First, I installed this kernel on my system. It took down the temperature a little bit but it’s not enough for me. So, I kept searching…

2) Next, I found a small app called Jupiter which can control your power management settings specially for the CPU. I installed it and (you need to reboot after installing) set the “Performance Mode” to “Power Saver” (Click on the lightning bolt icon at top panel to change these settings). This also helps me to bring down the heat a little more…

3) I’m desperate, so I tried one more trick which is a workaround for the ASPM issue. It’s a fix which you apply to the boot loader which will allow you to force ASPM. Try it : LINUX KERNEL POWER ISSUE / OVERHEAT WORKAROUND

After applying all these settings, I managed to get down the CPU heat to nearly 60 C. The VGA temperature also stays at 61 C. But, that’s when the machine idle. When I start my work (Normal browsing), the heat goes up to about 65 and 70!

So, did it fix the issue? Only a little. Still we don’t have an exact fix for this problem. Still Windows handles the heat better than Linux. But we can’t ditch Linux and go for Windows! Some say that a fix will be included in the Linux kernel 3.2. But still we don’t have any confirmation… I think they better fix it soon. Already a lot of people face this problem!

PS : Don’t worry your laptop won’t burn like the one showed in the top photo. But from the inside it can do a lot of damage if it keeps getting over heated!
