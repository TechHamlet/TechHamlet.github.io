Transferring From Windows to Ubuntu(Got to Hear My Experience)

Aug 16th, 2011 | Comments

I happen to try out Ubuntu in the virtualbox and I remember exactly I downloaded it when Ubuntu 11.04 is about to be released(What I downloaded was Ubuntu 10.10)… First time I downloaded the Server edition of Ubuntu not knowing that there was no GUI in it and you have to install the Gnome if you want to… From there on I wanted to try out Ubuntu non-virtualboxy to experience Ubuntu only.

I first carried out my mission in a desktop Computer. It was running Windows 7 and I installed Ubuntu 10.10 in it. It was good experience and I got know the difference between Ubuntu(ext4j partitions) and that Ubuntu loads Windows Partitions as Virtual Images. Now this actually was pretty annoying since I didn’t wanted icons to appear when I enter a different drive on my Desktop.

After a while I considered what I do with my computer and found out that everything I do except for one thing can be done in Ubuntu only environment(That one thing was using Photoshop). IDEs I wanted were fully supporting to Ubuntu and installing and setting Apache was pretty easy too.

While transferring from Windows 7 to Ubuntu I had to download Netbeans, Eclipse and my other development tools. What annoyed me most was installing of JDK. I tried to use

sudo apt-get install sun-java6-jdk sun-java6-jre

But the speed of downloading was pretty low. About 5 kbps so I had to quit using Apt to install JDK. I downloaded the JDK from Oracle site and then tried to install but the path was giving me a load of trouble. Last issue that arose to me was getting Eclipse to start. I had to wrap JDK path around Eclipse so I that Eclipse new where the JDK was. But after installing Netbeans the JDK problem solved easily.

For the Apt-get slowness I wanted to try out something. How about use a lot of servers and multiple connections to get the file needed. While searching for this I found the solution already built - apt-fast.

To use this you got to first go to the terminal and install the Axel download manager.

sudo apt-get install axel

After that add this repository and you’ll be able to install this normally. sudo add-apt-repository ppa:tldm217/tahutek.net sudo apt-get update sudo apt-get install apt-fast

While I was suffering from problems major sites that helped should be great reference for you if you are planning to migrate to Ubuntu and finding it darn hard.

  • First place to go is - Google. Google it and you’ll be able to find the solution quickly.

  • Next place to search is Ubuntu Forums. It’s a great place where experts as well as newbees are helping each other.

  • AskUbuntu is the stackexchange site that’ll help you when you can’t find the solution to your problem. You’ll get answers quickly. It’s a very good community.

Now for the software replacements for Ubuntu. You’ll need basic software to live in Ubuntu. Firefox is getting worse and worse so best option is to download Chrome from there repository. And for the download manager I suggest using SteadyFlow. It’s one of the solutions I tried and it’s working perfectly. For torrent downloading you are better of with Deluge torrent client. It’s very good plus there are less problems with it. For chat you can either go with Empathy or Pidgin but I prefer Pidgin over Empathy.

I hope good luck with your Ubuntu transfer and if you haven’t felt like transferring and just read the post go ahead and think what do you actually do on your computer. If gaming is not there in the list Ubuntu could be ideal for you. But you’ll find it hard and you might have to refer more than usual to work in Ubuntu for the first few days.

You can visit our Ubuntu Category and find more posts to try out and keep the fire going. So hope you’ll enjoy the power of Ubuntu.
