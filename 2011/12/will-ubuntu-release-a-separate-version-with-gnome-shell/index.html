Will Ubuntu Release a Separate Version With Gnome Shell?

Dec 19th, 2011 | Comments

In the past few weeks, I used Linux Mint simply because its Ubuntu without Unity! I wanted to work on a new project and wanted to build a custom environment to work on… So, I switched to Ubuntu and immediately installed Gnome Shell and started customizing! Wow… its even better than Mint!

I customized most of my desktop using Shell Extensions, Icons and Themes. As I said earlier, thing were better than in Mint. But, still, I don’t even want to switch back to Unity.

I know, most of our reader hate Unity and want Gnome shell. So, we wrote how to install Gnome shell on Ubuntu and trust me, in the past few months, mostly, that post is the most viewed page on this site! Another page pass it only during a special traffic spike. Otherwise, no one can beat it!

That proves how badly people want to move to Gnome Shell. At first, for me, it looked like both Unity and Gnome shell were hard to tame. But, with the release of Gnome Shell Extensions, Gnome Shell became more customizable than ever.

Now, we know the exact target of Unity. Ubuntu is targeting to build one desktop to fit even the mobile devices. They want to go into tablet and smart phone business! But, joining mobile UI and the desktop UI is not a good idea. Unity is a good example and I think Windows 8 is also going to make the same mistake.

But recently, Muktware.com have spotted an email sent by an Ubuntu developer asking for more contributors. The most important piece on that mail is the part that mentions about the GNOME session :

the GNOME session, it would be really cool if a team stepped up to maintain a derivative iso with gnome-shell by default

Still there are people who want to see Gnome Shell in Ubuntu in their development team. I think they must do this if they want to stay afloat! But if they do it, then, it will get more popular than the default Ubuntu! What do you think? Will they do it?
